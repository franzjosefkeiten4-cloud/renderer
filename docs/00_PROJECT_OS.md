# Keiten Experience Engine

# PROJECT OS

Version 0.1

---

# Mission

Wir entwickeln keine Sprachlern-App.

Wir entwickeln eine Experience Engine.

Menschen sollen Geschichten erleben.

Während sie Geschichten erleben,

entwickeln sie nahezu unbemerkt

• Sprachgefühl

• Wortschatz

• Grammatikgefühl

• kulturelles Verständnis

• Hörverständnis

• Lesefreude

• Mut zum Sprechen.

---

# Vision

Jeder Mensch besitzt natürliche Lernmechanismen.

Unsere Aufgabe besteht nicht darin,

Wissen einzufüllen.

Unsere Aufgabe besteht darin,

Situationen zu erschaffen,

in denen Lernen fast unvermeidlich wird.

---

# Leitprinzipien

1.
Geschichte vor Grammatik.

2.
Neugier vor Leistung.

3.
Sprachgefühl vor Regelwissen.

4.
Emotion vor Information.

5.
Entdecken statt Belehren.

6.
Keine Tests.

7.
Keine Bestrafung.

8.
Fehler sind Lernmaterial.

9.
Technik dient ausschließlich der Geschichte.

10.
Der Nutzer soll weiterlesen wollen.

11.
Der Lerner darf niemals interne Produktions- oder Technikdetails sehen.

---

# Pilotphilosophie

Pilotfeedback ist Gold.

Jede Rückmeldung wird ernst genommen.

Nicht weil der Nutzer immer Recht hat.

Sondern weil jedes Missverständnis

eine Schwachstelle der Experience zeigt.

---

# Entscheidungsregel

Wenn Technik

und Geschichte

miteinander konkurrieren,

gewinnt fast immer

die Geschichte.

Wenn Didaktik

und Geschichte

miteinander konkurrieren,

gewinnt die Geschichte,

solange das Lernen erhalten bleibt.

---

# Unsere wichtigste Kennzahl

Nicht:

Wie viele Wörter wurden gelernt?

Sondern:

Wie oft möchte jemand weiterlesen?

---

# Qualitätskriterien

Eine Episode ist erst fertig,

wenn sie folgende Fragen besteht.

✓ Ist sie verständlich?

✓ Macht sie neugierig?

✓ Klingt sie menschlich?

✓ Entstehen Bilder?

✓ Möchte man weiterlesen?

✓ Unterstützt jede Interaktion die Geschichte?

✓ Fühlt sie sich nicht wie Unterricht an?
